
# 🏦 iM뱅크 전화 상담 AI 요약·챗봇 자동 등록 시스템

> **본부–영업점 전화 상담을 자동으로 기록·분석하여 실무 지식 데이터베이스를 구축하는 AI 기반 정보계 시스템**

---

## 📌 프로젝트 개요

iM뱅크 영업점 직원들은 업무 처리 과정에서 본부 부서에 전화로 문의하는 경우가 빈번하며, 그 중 상당수는 반복적인 질문으로 구성되어 있습니다.  
그러나 이러한 상담 내용은 담당자의 기억이나 개인 메모에 의존할 뿐, 체계적으로 기록·축적되지 않아 조직 차원의 지식 자산으로 활용되지 못하고 있었습니다.

**iM뱅크 본부 상담 지식 백서 시스템**은 본부와 영업점 간 전화 상담 내용을 AI 기술로 자동 수집·분석하여, 실제 현장 상담 사례 기반의 **정보계 지식 백서 데이터베이스**를 구축합니다.  
축적된 지식은 전 직원이 검색·참조할 수 있는 실무형 지식 자산으로 체계화되며, **사내 챗봇 시스템과 연동하여 상담 사례가 즉시 챗봇 Q&A로 반영**됩니다. 이를 통해 영업점 직원들이 본부에 전화하지 않고도 챗봇으로 바로 답변을 얻을 수 있는 환경을 구축합니다.

---

## 🚨 문제 인식

| 문제 | 내용 |
|------|------|
| 📞 반복적인 전화 문의 | 방대한 양의 업무 질문이 반복적으로 본부로 접수되어 업무 비효율 발생 |
| 🗂️ 지식의 단절 | 상담 내용이 개인에게 귀속되어 조직 전체의 지식으로 공유되지 않음 |
| ⏳ 기록 부재 | 상담 후 내용이 텍스트로 정리되지 않아 사례 축적이 불가능 |
| 🔍 정보 접근 한계 | 유사 사례를 검색할 수 있는 체계적 시스템이 부재하여 같은 문의 반복 |
| 🤖 챗봇 업데이트 한계 | 사내 챗봇이 운영 중이나 새로운 상담 사례가 즉각 반영되지 않아 직원들이 결국 본부에 전화로 문의 |

---

### 기존 지식 등록 시스템에 대하여
 
현재 사내에는 직원들이 질문과 답변을 직접 등록할 수 있는 **지식 제안 시스템이 이미 운영**되고 있습니다.  
조직 내 지식 공유의 필요성을 일찍이 인식하고 구축한 의미 있는 시스템입니다.
 
다만 실무 현장에서는 다음과 같은 현실적인 이유로 적극적인 활용이 쉽지 않은 상황입니다.
 
- 상담 후 내용을 **별도로 정리하고 직접 입력**해야 하는 추가 업무 부담
- 바쁜 업무 중 등록 과정에 시간을 할애하기 어려운 현실적 제약
- 결과적으로 **유용한 상담 사례가 등록되지 못한 채 소실**
 
즉, 시스템의 부재가 아니라 **등록 과정의 번거로움**이 지식 축적을 가로막는 핵심 원인입니다.  
본 프로젝트는 이 병목 구간을 AI로 자동화하여, **기존 시스템이 본래 목적대로 충분히 활용될 수 있도록 지원**하는 것을 목표로 합니다.

## 💡 해결 방안
 
본부–영업점 전화 상담 내용을 **AI 음성 인식(STT)** 기술로 자동 텍스트화하고, AI 기반 요약·구조화 기능을 통해 실무 지식 백서 형태로 자동 정리합니다.  
정리된 상담 요약 및 Q&A 초안은 담당자의 간단한 검토·수정 후 **지식 백서 데이터베이스에 즉시 등록**되어 전사적으로 공유되며, **사내 챗봇 시스템에도 바로 반영**되어 영업점 직원들이 전화 문의 없이 챗봇으로 즉시 답변을 받을 수 있게 됩니다.

## 📈 기대 효과

```
반복 전화 문의 감소
  └─ 자주 묻는 사례가 백서에 빠르게 축적 → 직원 자기해결율 향상

본부 업무 효율성 향상
  └─ 반복 상담 응대 감소 → 본연의 업무 집중 가능

지식 관리 체계 고도화
  └─ 실제 상담 사례 기반의 현장 밀착형 실무 지식 데이터 축적

영업점 업무 처리 속도 개선
  └─ 지식 백서 검색을 통한 즉각적인 정보 접근 가능

AI 기반 업무 지원 환경 구축
  └─ 조직 내 지식 공유 및 업무 자동화 수준 향상

챗봇 지식 자동 업데이트
  └─ 상담 사례가 챗봇 Q&A로 자동 변환 → 챗봇 활용도 향상 → 전화 문의 감소 선순환
기존 지식 제안 시스템 활성화
  └─ AI 자동화로 등록 부담 제거 → 기존 시스템이 본래 목적대로 운영
```
