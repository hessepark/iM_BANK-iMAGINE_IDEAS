# iMGuard — 창구 인감·서명 스마트 검증 시스템
> AI 기반 인감·서명 자동 검증 시스템 (Smart Signature & Seal Verification System)

---

## 1. 프로젝트 개요

은행 창구에서 출금·제신고 처리 시 인감·서명 일치 여부를 **직원이 육안으로 판단**하던 기존 방식을 탈피하여,  
AI와 시스템이 자동으로 검증하는 **창구 사고 예방 시스템**이다.

기존에는 직원이 직접 전산 조회 화면에서 인감·서명 변경 이력을 일일이 조회하고,  
과거부터 순차적으로 확인하여 최신 인감·서명을 찾아야 했다.  
본 시스템은 이 과정을 자동화하여 **통장 입력 즉시 최신 인감·서명 이미지를 화면에 자동으로 띄워주고**,  
AI가 전자문서와 비교하여 일치 여부를 판정한다.

또한 기존에는 직원이 고개를 숙여 통장 실물을 직접 확인해야 했지만,  
본 시스템 도입 후에는 **통장은 계좌 인식용으로만 사용**하고  
이후 모든 검증은 **화면만 보고 처리**할 수 있어 업무 집중도와 처리 속도가 크게 향상된다.

> "인감·서명 검증을 직원의 경험이 아닌 AI와 시스템으로 전환하여 금융 사고를 예방하는 창구 혁신 시스템"

---

## 2. 문제인식

현재 은행 창구 업무에서는 출금 및 제신고 처리 시 고객의 인감 또는 서명 일치 여부를 직원이 육안으로 판단하고 있으며, 다음과 같은 문제가 지속적으로 발생하고 있다.

- 인감통장 / 서명통장 / 인감·서명 통장 **유형 혼동**
- 통장 실물과 전산 화면 간 **불일치 발생**
- 최신 인감·서명 **변경 이력 확인 누락**
- BPR, 물류, 제신고서 등 **과거 자료를 최신으로 오인**하는 실수
- 무인감 통장 이월 재발행, 통장 인감 분실 재발행, 서명 변경 등 **복잡한 조회 과정**
- 신입 직원의 업무 숙련도 부족으로 인한 **판단 오류**

특히 **무인감 통장 이월 재발행**의 경우, 통장 실물만으로는 확인이 불가능하여 반드시 전산 조회가 필요하다.

> 💡 **무인감 통장 이월 재발행이란?**  
> ATM·CD기에서 통장을 새로 발급받을 때, 기계 특성상 도장(인감)을 찍을 수 없어  
> **도장이나 서명이 없는 상태로 통장이 발행**되는 것을 말한다.  
> 이 경우 통장 실물에는 아무 도장도 없기 때문에, 직원이 실물만 보고는 인감을 확인할 수 없고  
> 반드시 전산에서 최신 등록 인감·서명을 별도로 조회해야 한다.

현재는 전산 조회 화면에서 특정 코드를 통해 이력을 순차적으로 확인해야 하며, 이 과정에서 시간 지연과 오류가 빈번하게 발생한다.

> 단순한 실수 문제를 넘어 → **복잡한 조회 구조 자체**가 업무 시간 증가와 오류 발생의 주요 원인

## 3. 해결방안

고객이 통장을 입력하는 순간, 시스템이 자동으로 다음 정보를 조회한다.

- 계좌의 현재 등록 유형 (인감 / 서명 / 인감·서명)
- 최신 등록 인감 및 서명 이미지

  | 등록 인감 (예시) | 등록 서명 (예시) |
  |:---:|:---:|
  | <img src="https://github.com/user-attachments/assets/c2137189-3aca-48b0-94a1-597c0372eb56" width="100"/> | <img src="https://github.com/user-attachments/assets/b716c723-d558-4936-86ad-51fc655c8d8c" width="200"/> |

- 최근 인감 분실, 서명 변경, 통장 재발행 이력

이후 전자문서에 포함된 도장 또는 서명을 자동으로 인식하고, 등록된 원본 데이터와 비교하여 AI가 일치 여부를 판단한다.  
계좌 유형에 따라 입력 가능한 방식이 자동으로 제어되며, AI는 비교 결과를 3단계로 제공한다.

| 단계 | 색상 | 의미 |
|------|------|------|
| 1단계 | 🟢 초록 | 일치 (안정) — 전송 가능 |
| 2단계 | 🟡 주황 | 유사 (재확인 필요) — 확인 후 전송 |
| 3단계 | 🔴 빨강 | 불일치 (위험) — 전송 차단 |

## 4. 주요기능

**계좌 유형 자동 인식 및 UI 제어**
- 인감 / 서명 / 인감·서명 유형 자동 표시
- 유형에 따라 입력 버튼 자동 활성/비활성

**최신 인감·서명 자동 조회**
- 계좌 기준 최신 등록 원본 이미지 자동 호출
- 과거 자료 사용 방지

**전자문서 기반 자동 인식 및 비교**
- 전자문서 내 도장 및 서명 영역 자동 탐지
- AI 기반 이미지 비교 수행

**AI 일치율 분석 및 3단계 판정**
- 일치율 % 수치 제공
- 초록 / 주황 / 빨강 단계로 직관적 표시

**전송 전 자동 검증 기능**
- 불일치 또는 위험 상태 시 전송 차단 또는 경고

**변경 이력 자동 반영**
- 인감 분실, 서명 변경, 통장 재발행 이력 자동 확인

**연동정보 입력 체크**
- 연동정보란이란, 출금 시 다른 은행 계좌로 동시에 이체를 원하는 경우 직원이 전산에 입력하는 항목이다
- 전산에 연동정보가 입력되어 있음에도 고객이 전자문서(출금전표)의 해당란을 작성하지 않은 경우, 전송 전 경고 팝업을 통해 누락을 안내하고 전송을 차단한다

**데모 설명**
